**Задача:** Реализация Dictionary API (Справочники).

Чтобы избежать рассинхрона между фронтендом и бэкендом, реализовал паттерн "Single Source of Truth".
Вынес списки тем и сложностей в отдельный файл.
Создал эндпоинт `GET /api/dictionaries`, который отдает эти списки фронтенду.
Сначала описал контракт в `api.yaml`, затем реализовал код.

**Итог:** Инфраструктура для динамического конфигурирования готова. Следующий шаг — использовать эти же константы для валидации входящих запросов в чат.
